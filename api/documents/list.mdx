---
title: "List and retrieve documents via the API"
description: "Fetch your document library with GET /documents, get per-status counts with GET /documents/counts, and retrieve a single document with its full node tree."
---

Use these endpoints to read documents from your DocInject organization. You can list all documents you own or that have been shared with you, get a breakdown of document counts by status, or fetch a single document with its complete section and step hierarchy.

## List documents

```
GET https://api.docinject.io/api/v1/documents
```

Returns an array of documents. By default, only documents you own are returned. Pass `filter=shared` to list documents where you are assigned as editor instead.

### Query parameters

<ParamField query="filter" type="string" default="owned">
  Which documents to return. `owned` returns documents you created. `shared` returns documents where you are assigned as editor.

  Accepted values: `owned`, `shared`
</ParamField>

<ParamField query="limit" type="number">
  Maximum number of documents to return. Omit to return all documents.
</ParamField>

### Example request

<CodeGroup>

```bash curl
curl "https://api.docinject.io/api/v1/documents?filter=owned&limit=25" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const res = await fetch(
  'https://api.docinject.io/api/v1/documents?filter=owned&limit=25',
  {
    headers: {
      Authorization: `Bearer ${process.env.DOCINJECT_API_KEY}`,
    },
  }
);
const documents = await res.json();
```

```python Python
import os, requests

res = requests.get(
    'https://api.docinject.io/api/v1/documents',
    params={'filter': 'owned', 'limit': 25},
    headers={'Authorization': f'Bearer {os.environ["DOCINJECT_API_KEY"]}'},
)
documents = res.json()
```

</CodeGroup>

### Response

Returns `200 OK` with an array of document objects.

<ResponseField name="id" type="string" required>
  Unique identifier for the document.
</ResponseField>

<ResponseField name="organization_id" type="string" required>
  ID of the organization that owns this document.
</ResponseField>

<ResponseField name="created_by" type="string" required>
  User ID of the document owner.
</ResponseField>

<ResponseField name="editor_id" type="string">
  User ID of the assigned editor, if any.
</ResponseField>

<ResponseField name="parent_doc_id" type="string">
  ID of the previous published version this document was revised from. Present only on revision drafts.
</ResponseField>

<ResponseField name="title" type="string" required>
  Document title.
</ResponseField>

<ResponseField name="version" type="string" required>
  Version label, for example `1.0` or `Q2-2025`.
</ResponseField>

<ResponseField name="department" type="string" required>
  Department or team this document belongs to.
</ResponseField>

<ResponseField name="status" type="string" required>
  Current lifecycle status: `draft`, `published`, or `archived`.
</ResponseField>

<ResponseField name="published_at" type="string">
  ISO 8601 timestamp of when the document was last published. `null` for drafts.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the document was created.
</ResponseField>

<ResponseField name="updated_at" type="string" required>
  ISO 8601 timestamp of the last update.
</ResponseField>

<ResponseField name="nodes" type="object[]">
  Full node tree. This field is always an empty array on list responses — fetch a single document to get the complete tree.
</ResponseField>

## Get document counts

```
GET https://api.docinject.io/api/v1/documents/counts
```

Returns a count of your documents broken down by status. Use this to build dashboards or status summaries without fetching the full document list.

### Example request

<CodeGroup>

```bash curl
curl "https://api.docinject.io/api/v1/documents/counts" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const res = await fetch('https://api.docinject.io/api/v1/documents/counts', {
  headers: { Authorization: `Bearer ${process.env.DOCINJECT_API_KEY}` },
});
const counts = await res.json();
```

</CodeGroup>

### Response

Returns `200 OK` with an object containing counts for each status.

<ResponseField name="draft" type="number" required>
  Number of documents in draft status.
</ResponseField>

<ResponseField name="published" type="number" required>
  Number of published documents.
</ResponseField>

<ResponseField name="archived" type="number" required>
  Number of archived documents.
</ResponseField>

```json 200
{
  "draft": 4,
  "published": 17,
  "archived": 3
}
```

## Get a single document

```
GET https://api.docinject.io/api/v1/documents/{doc_id}
```

Returns a single document including its complete node tree — sections, steps, and sub-steps — sorted by `order`.

### Path parameters

<ParamField path="doc_id" type="string" required>
  The ID of the document to retrieve.
</ParamField>

### Example request

<CodeGroup>

```bash curl
curl "https://api.docinject.io/api/v1/documents/doc_01hxyz1a2b3c4d5e6f7g8h9j" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const docId = 'doc_01hxyz1a2b3c4d5e6f7g8h9j';
const res = await fetch(
  `https://api.docinject.io/api/v1/documents/${docId}`,
  {
    headers: { Authorization: `Bearer ${process.env.DOCINJECT_API_KEY}` },
  }
);
const doc = await res.json();
```

```python Python
import os, requests

doc_id = 'doc_01hxyz1a2b3c4d5e6f7g8h9j'
res = requests.get(
    f'https://api.docinject.io/api/v1/documents/{doc_id}',
    headers={'Authorization': f'Bearer {os.environ["DOCINJECT_API_KEY"]}'},
)
doc = res.json()
```

</CodeGroup>

### Response

Returns `200 OK` with a document object. The `nodes` field contains the full content tree — each node may have a `children` array with nested nodes of the next level.

<ResponseField name="nodes" type="object[]">
  Ordered array of top-level section nodes. Each section contains `children` (steps), and each step may contain `children` (sub-steps).

  <Expandable title="node properties">
    <ResponseField name="id" type="string" required>
      Unique node identifier.
    </ResponseField>
    <ResponseField name="document_id" type="string" required>
      ID of the parent document.
    </ResponseField>
    <ResponseField name="parent_id" type="string">
      ID of the parent node. `null` for top-level sections.
    </ResponseField>
    <ResponseField name="node_type" type="string" required>
      Type of node: `section`, `step`, or `sub_step`.
    </ResponseField>
    <ResponseField name="title" type="string" required>
      Node title.
    </ResponseField>
    <ResponseField name="content" type="object">
      Rich text content stored as structured JSON. `null` if no content has been added.
    </ResponseField>
    <ResponseField name="order" type="number" required>
      Sort position among siblings. Nodes are returned pre-sorted by this value.
    </ResponseField>
    <ResponseField name="created_by" type="string" required>
      User ID of the member who created this node.
    </ResponseField>
    <ResponseField name="created_at" type="string" required>
      ISO 8601 creation timestamp.
    </ResponseField>
    <ResponseField name="updated_at" type="string" required>
      ISO 8601 last-updated timestamp.
    </ResponseField>
    <ResponseField name="children" type="object[]" required>
      Nested child nodes, sorted by `order`. Empty array if there are no children.
    </ResponseField>
  </Expandable>
</ResponseField>
