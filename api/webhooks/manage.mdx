---
title: "Manage webhooks via the DocInject API"
description: "Register, update, test, and delete webhook configurations for your DocInject organization using the REST API. Includes the event delivery log endpoint."
---

Use the webhooks API to manage the endpoints DocInject delivers events to. All endpoints require a valid bearer token and a Scale plan subscription. Webhook owners and organization admins can modify or delete a given configuration; any member can trigger a test delivery.

Base URL: `https://api.docinject.io/api/v1`

---

## Webhook config object

Most endpoints return a webhook config object. Its shape is consistent across list, create, and update responses.

```json
{
  "id": "wh-uuid-0001",
  "organization_id": "a1b2c3d4-0000-0000-0000-111122223333",
  "created_by": "user-uuid-0001",
  "name": "Publish notifier",
  "url": "https://hooks.acme.example/docinject",
  "event_types": ["document.published", "document.revised"],
  "is_active": true,
  "created_at": "2024-03-01T08:00:00Z"
}
```

<Note>
  The `secret` field is returned **only** in the response to `POST /webhooks`. Store it immediately — it cannot be retrieved again.
</Note>

## List webhooks

Retrieve all webhook configurations registered for your organization.

```
GET /webhooks
```

**Response** — `200 OK`: an array of [webhook config objects](#webhook-config-object).

<CodeGroup>
```bash curl
curl https://api.docinject.io/api/v1/webhooks \
  -H "Authorization: Bearer <token>"
```

```javascript JavaScript
const res = await fetch("https://api.docinject.io/api/v1/webhooks", {
  headers: { Authorization: "Bearer <token>" },
});
const webhooks = await res.json();
```
</CodeGroup>

## Register a webhook

Create a new webhook endpoint. Requires the Scale plan.

```
POST /webhooks
```

**Request body**

<ParamField body="name" type="string" required>
  A human-readable label for this webhook (e.g. `"Slack publish alerts"`).
</ParamField>

<ParamField body="url" type="string" required>
  The HTTPS URL DocInject will POST events to.
</ParamField>

<ParamField body="event_types" type="string[]" required>
  One or more event type strings to subscribe to. Must be values from the [event reference](/api/webhooks/events). Passing an unrecognized value returns `400`.
</ParamField>

**Response** — `201 Created`: a webhook config object with `secret` included.

<ResponseField name="id" type="string">UUID of the new webhook configuration.</ResponseField>
<ResponseField name="organization_id" type="string">UUID of your organization.</ResponseField>
<ResponseField name="created_by" type="string">UUID of the user who registered this webhook.</ResponseField>
<ResponseField name="name" type="string">The name you provided.</ResponseField>
<ResponseField name="url" type="string">The endpoint URL.</ResponseField>
<ResponseField name="event_types" type="string[]">The subscribed event types.</ResponseField>
<ResponseField name="is_active" type="boolean">Always `true` on creation.</ResponseField>
<ResponseField name="created_at" type="string">ISO 8601 creation timestamp.</ResponseField>
<ResponseField name="secret" type="string">
  HMAC-SHA256 signing secret. Returned **once only** — store this value immediately. Use it to verify the `X-DocInject-Signature` header on incoming requests. See the [webhooks overview](/api/webhooks/overview#verifying-signatures) for verification details.
</ResponseField>

<CodeGroup>
```bash curl
curl -X POST https://api.docinject.io/api/v1/webhooks \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Publish notifier",
    "url": "https://hooks.acme.example/docinject",
    "event_types": ["document.published"]
  }'
```

```javascript JavaScript
const res = await fetch("https://api.docinject.io/api/v1/webhooks", {
  method: "POST",
  headers: {
    Authorization: "Bearer <token>",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    name: "Publish notifier",
    url: "https://hooks.acme.example/docinject",
    event_types: ["document.published"],
  }),
});
const { secret, ...config } = await res.json();
// Store `secret` securely — it will not be returned again.
```
</CodeGroup>

**Example response**

```json
{
  "id": "wh-uuid-0001",
  "organization_id": "a1b2c3d4-0000-0000-0000-111122223333",
  "created_by": "user-uuid-0001",
  "name": "Publish notifier",
  "url": "https://hooks.acme.example/docinject",
  "event_types": ["document.published"],
  "is_active": true,
  "created_at": "2024-03-01T08:00:00Z",
  "secret": "a3f9c2e1b8d7..."
}
```

## Update a webhook

Modify an existing webhook configuration. Send only the fields you want to change. The webhook owner or any org admin can update.

```
PATCH /webhooks/{config_id}
```

**Path parameters**

<ParamField path="config_id" type="string" required>
  UUID of the webhook configuration to update.
</ParamField>

**Request body** — all fields optional

<ParamField body="name" type="string">
  New label for the webhook.
</ParamField>

<ParamField body="url" type="string">
  New destination URL.
</ParamField>

<ParamField body="event_types" type="string[]">
  Replacement list of event types. Overwrites the existing list entirely.
</ParamField>

**Response** — `200 OK`: the updated webhook config object (without `secret`).

<CodeGroup>
```bash curl
curl -X PATCH https://api.docinject.io/api/v1/webhooks/wh-uuid-0001 \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "event_types": ["document.published", "document.revised", "member.joined"]
  }'
```

```javascript JavaScript
const res = await fetch(
  "https://api.docinject.io/api/v1/webhooks/wh-uuid-0001",
  {
    method: "PATCH",
    headers: {
      Authorization: "Bearer <token>",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({
      event_types: ["document.published", "document.revised", "member.joined"],
    }),
  }
);
const updated = await res.json();
```
</CodeGroup>

## Delete a webhook

Permanently remove a webhook configuration. The webhook owner or any org admin can delete.

```
DELETE /webhooks/{config_id}
```

**Path parameters**

<ParamField path="config_id" type="string" required>
  UUID of the webhook configuration to delete.
</ParamField>

**Response** — `204 No Content`.

<CodeGroup>
```bash curl
curl -X DELETE https://api.docinject.io/api/v1/webhooks/wh-uuid-0001 \
  -H "Authorization: Bearer <token>"
```

```javascript JavaScript
await fetch("https://api.docinject.io/api/v1/webhooks/wh-uuid-0001", {
  method: "DELETE",
  headers: { Authorization: "Bearer <token>" },
});
```
</CodeGroup>

## Test a webhook

Send a test ping to a registered endpoint to confirm it is reachable. Any org member can trigger a test. DocInject sends a single request with `event_type: "webhook.test"` and waits up to 10 seconds for a response.

```
POST /webhooks/{config_id}/test
```

**Path parameters**

<ParamField path="config_id" type="string" required>
  UUID of the webhook configuration to test.
</ParamField>

**Response** — `200 OK`

<ResponseField name="status" type='"delivered" | "failed"'>
  `"delivered"` if the endpoint responded with a 2xx status within 10 seconds; `"failed"` otherwise.
</ResponseField>

<ResponseField name="http_status" type="integer | null">
  The HTTP status code returned by your endpoint, or `null` if no response was received (e.g. connection timeout).
</ResponseField>

<CodeGroup>
```bash curl
curl -X POST https://api.docinject.io/api/v1/webhooks/wh-uuid-0001/test \
  -H "Authorization: Bearer <token>"
```

```javascript JavaScript
const res = await fetch(
  "https://api.docinject.io/api/v1/webhooks/wh-uuid-0001/test",
  {
    method: "POST",
    headers: { Authorization: "Bearer <token>" },
  }
);
const { status, http_status } = await res.json();
// { status: "delivered", http_status: 200 }
```
</CodeGroup>

**Example responses**

```json Delivered
{ "status": "delivered", "http_status": 200 }
```

```json Failed (non-2xx)
{ "status": "failed", "http_status": 500 }
```

```json Failed (no response)
{ "status": "failed", "http_status": null }
```

## Get event delivery log

Retrieve the last 100 outbound webhook events for your organization, ordered by most recent first. Use this to inspect delivery status and retry counts.

```
GET /webhooks/events
```

**Response** — `200 OK`: an array of event log objects.

<ResponseField name="id" type="string">UUID of the event log entry.</ResponseField>
<ResponseField name="event_type" type="string">The event type that was dispatched (e.g. `"document.published"`).</ResponseField>
<ResponseField name="direction" type='"outbound"'>Always `"outbound"` for events in this log.</ResponseField>
<ResponseField name="status" type='"pending" | "delivered" | "failed"'>
  Current delivery status. `"pending"` means delivery is still in progress.
</ResponseField>
<ResponseField name="attempts" type="integer">Number of delivery attempts made so far. Maximum is `3`.</ResponseField>
<ResponseField name="created_at" type="string">ISO 8601 timestamp when the event was first enqueued.</ResponseField>

<CodeGroup>
```bash curl
curl https://api.docinject.io/api/v1/webhooks/events \
  -H "Authorization: Bearer <token>"
```

```javascript JavaScript
const res = await fetch("https://api.docinject.io/api/v1/webhooks/events", {
  headers: { Authorization: "Bearer <token>" },
});
const events = await res.json();
```
</CodeGroup>

**Example response**

```json
[
  {
    "id": "evt-uuid-0001",
    "event_type": "document.published",
    "direction": "outbound",
    "status": "delivered",
    "attempts": 1,
    "created_at": "2024-03-10T11:00:05Z"
  },
  {
    "id": "evt-uuid-0002",
    "event_type": "member.invited",
    "direction": "outbound",
    "status": "failed",
    "attempts": 3,
    "created_at": "2024-03-09T14:22:10Z"
  }
]
```
